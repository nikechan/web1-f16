Using Fonts on the Web

Fallback fonts

We should be seeing paragraphs ( p), like this one styled in Futura. We use a CSS like like: font-family: Futura, "Trebuchet MS", Arial, sans-serif; this, to provide fallback fonts. Each font is tried in order, from left-to-right, until one is found. “sans-serif” is a required default, it is usually Arial on Windows and Helvetica on OS-X.

WebFonts: Ostrich Sans (self-hosted)

WebFonts allow us greater control over which font is actually displayed for our user. If the user doesn't have the font installed, we provide it too them. There are two main ways of doing this. We'll call them self-hosted and service-hosted. Self-hosted just means that you need to upload all the font files to your own web server.

Ostrich Sans is provided freely by the League of Moveable Type. In order to use it we need to download it from their website at Ostrich Sans. Unzip and copy to our web project. Then we need to include a link to their CSS file from our HTML file. Then we need to actually specify which elements should be styled using this font in our CSS. All our h1 tags should be styled with Ostrich Sans. Like this one:

I am in Ostrich Sans

WebFonts: Roboto (service-hosted)

If someone else hosts the font for us, then we can skip the step of downloading it. Services like fonts.com, Typekit and Google Fonts do just that. We'll use Google Fonts since it is free. Essentially you just need to pick a font, copy their code into your HTML file (to include the font), then use the font for an element in your CSS.
