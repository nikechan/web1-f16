Homework #2 —Event Promotion

  • Course Home

Create a promotional single-page website for a real or aspirational multi-day event. Possible choices could be: conferences, multi-artist music, visual or performing art festivals or an artist talk series. The website could be promoting a future event or documenting a past event. You will continue to work on this site throughout the semester, choose a topic you will be excited about. Due by the start of next class.

Adapted from Saul Baizman's The Conference assignment.

Goals

  1. Practice using a variety of HTML tags in a semantic fashion
  2. Practice using basic CSS styling to influence the visual page aesthetics
  3. Practice designing site content in order to communicate information to a variety of audiences
  4. Practice using GitHub Desktop
  5. Practice editing code using the Textmate text editor

Deliverables

A single-page event website synced to GitHub and linked to from your Student Page. See andrewringler.github.io/web1-f16-student/ (GitHub) for an example of how to do this. Should contain any information visitors to the event would need to know structured in a usable fashion.

Suggested content includes (you must include at least five of the following):

  1. One or more tables with a schedule of presentations, workshops, including the day, time, location, title of presentation, and name of presenter
  2. Artist/performer biographies
  3. Description of the event
  4. Registration, cost, attendance information
  5. Directions, location of the event
  6. At least one photos or graphic (possibly from a previous year's event)
  7. A menu
  8. A footer
  9. Content separated by sections

Minimal CSS styling including font choice, type sizes, type weight and color.

Fonts

A good list of fonts that are generally available on a wide range of computers are at CSS Font Stack. If you click on a font, like Optima, it gives you a font-family line that you can cut-paste into your CSS. Their h1 examples looks like: 

h1 {
   font-family: Optima, Segoe, "Segoe UI", Candara, Calibri, Arial, sans-serif;
}

This is specifying “fallbacks”, so first your browser will try to load the site with Optima, if not available, it will try Segoe, then Segoe UI, etc…. This is an opinated list of fonts that cssfontstack thoughts were similar-ish to Optima. Next week we will learn to use WebFonts, which will allow us to just provide the user with Optima if they don't have it.
